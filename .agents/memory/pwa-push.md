# PWA injectManifest + Push Notifications

## Rule
Use `strategies: "injectManifest"` in VitePWA config with `srcDir: "src"`, `filename: "sw.ts"` to support push events in the SW.

**Why:** `generateSW` (default) auto-generates the SW from workbox config only — no place to add custom `push` or `notificationclick` event listeners. `injectManifest` uses your own `src/sw.ts` and injects the precache manifest at build time.

**How to apply:**
- `src/sw.ts` must import from `workbox-precaching` (not routing/strategies — those need separate install)
- Install `workbox-precaching workbox-routing workbox-strategies` in the frontend package
- In `sw.ts`, use `/// <reference lib="webworker" />` and `declare const self: ServiceWorkerGlobalScope`
- `devOptions.enabled: false` prevents workbox-window errors in dev (SW only active in prod build)
- `registerSW` import from `virtual:pwa-register` should be guarded by `import.meta.env.PROD`

## VAPID Keys
- Generated via `node -e "const wp = require('web-push'); const k = wp.generateVAPIDKeys(); console.log(k)"`
- Set via `setEnvVars` (not secrets, since they're generated by us): `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_CONTACT`
- Public key returned by `GET /api/push/vapid-public-key` (no auth required — needed by frontend to subscribe)

## PWAInstallModal behavior
- Must check `window.self !== window.top` (iframe) → skip (Replit preview is an iframe)
- Must check `display-mode: standalone` → skip (already installed)
- iOS Safari → show guide after 2s delay
- Android/Chrome → wait for `beforeinstallprompt`, show modal
- localStorage key `merced_pwa_install`: `{installed: true}` never shows again; `{dismissedUntil: timestamp}` shows again after 3 days
- Mounted in `App.tsx` outside route switch so it appears on all pages

## push_subscriptions table
```sql
CREATE TABLE IF NOT EXISTS push_subscriptions (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  endpoint TEXT NOT NULL UNIQUE,
  p256dh TEXT NOT NULL,
  auth TEXT NOT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT NOW()
);
```

## Uint8Array TypeScript fix
Use `new ArrayBuffer(n)` then `new Uint8Array(buffer)` to get `Uint8Array<ArrayBuffer>` (not `Uint8Array<ArrayBufferLike>`), required by `pushManager.subscribe` applicationServerKey type.
